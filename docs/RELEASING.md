# Releasing equiv (prebuilt binaries via dist / cargo-dist)

Prebuilt binaries let the `equiv review` Action **download** the tool instead of
compiling Rust on every PR run (the keyless round-trip on PR #1 spent ~90s
mostly compiling). On a version tag, `dist` builds cross-platform binaries,
creates a GitHub Release and attaches the archives, installers and checksums.

## Config
- dist config lives in `[workspace.metadata.dist]` in the root `Cargo.toml`.
- `.github/workflows/release.yml`: generated by `dist generate`. **Do not
  hand-edit**; CI runs `dist generate --check` and fails if it drifts. Regenerate
  after any config change:
  ```
  dist generate           # from the repo root
  git add Cargo.toml .github/workflows/release.yml
  ```
- Targets: macOS arm64/x64, Linux x64/arm64 (gnu), Windows x64.

## Cut a release
1. Bump the version in `equiv/Cargo.toml` (`[workspace.package] version`).
2. Commit, then tag and push:
   ```
   git tag v0.1.0
   git push origin v0.1.0
   ```
3. The `release.yml` workflow builds every target and publishes a GitHub Release
   with assets named `equiv-cli-<target>.tar.xz` (`.zip` on Windows), each
   containing the `equiv` binary, plus `equiv-cli-installer.sh` / `.ps1` and
   `.sha256` checksums.

## How the review Action consumes it
`action.yml`'s "Resolve equiv binary" step downloads
the action repo's latest release asset `equiv-cli-x86_64-unknown-linux-gnu.tar.xz`,
extracts `equiv`, then sets `EQUIV_BIN`. **If no release exists yet (or the
download fails), it falls back to building from source**, so the Action works
before the first release and on forks, just slower. After the first tagged
release, PR runs skip compilation.

## Install for humans
Once a release exists:
```
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/Neelagiri65/equiv/releases/latest/download/equiv-cli-installer.sh | sh
```
